## Token用量查询

实时跟踪对话中的token消耗，支持手动查看、自动统计和配置界面设置。

### 功能特性

* **实时跟踪**：自动记录每次LLM响应的token消耗
* **手动统计**：输入`/token`查看当前对话段token统计信息，每次查看后自动重置
* **自动统计**：如果超过设定时间未使用`/token`，下一次AI回复后会自动输出统计信息
* **配置界面**：支持在AstrBot插件设置界面调整自动统计间隔
* **独立统计**：token统计信息与会话数据是相互独立的，重置会话并不影响当前的token统计
* **内存安全**：设定时间内无活动将自动清理会话，避免内存泄漏

### 安装方法

* 直接在AstrBot插件市场搜索插件名后自动安装

### 配置说明

#### 图形化配置界面
1. 在AstrBot管理界面进入"插件管理"
2. 找到"Token Tracker"插件
3. 点击"设置"或"配置"
4. 可配置以下选项：

   - **自动统计间隔（小时）**：超过此时间未使用`/token`，下一次AI回复后会自动输出统计信息
     - 范围：1-720小时（1小时到30天）
     - 默认：24小时
     - 支持滑块调整
  
   - **会话过期时间（小时）**：会话无活动超过此时间会被自动清理，避免内存泄漏
     - 范围：1-720小时
     - 默认：72小时
     - 支持滑块调整

5. 保存配置，修改会立即生效

#### 配置结构
插件使用`_conf_schema.json`文件定义配置结构，AstrBot会自动解析并在管理界面生成配置表单。#### 插件设置界面
1. 在AstrBot管理界面进入"插件管理"
2. 找到"Token Tracker"插件
3. 点击"设置"或"配置"
4. 修改"自动统计间隔（小时）"（范围：1-720小时，默认24小时）
5. 保存配置，插件会自动重新加载


### 使用说明

#### 基本使用
- 正常对话时，插件会自动记录token消耗
- 输入 `/token` 查看当前对话段的统计信息
- 查看后统计会自动重置，开始新的统计周期

#### 自动统计功能
- 默认情况下，如果24小时内未使用`/token`指令
- 用户发送消息并得到AI回复后，会自动输出统计信息
- 统计重置，重新开始计时

### 示例对话

```
用户：今天天气怎么样？
Bot：今天天气晴朗，温度适宜...

用户：/token
Bot：📊 本段对话Token统计：
    • 请求次数：1次
    • 输入Token：15个
    • 输出Token：20个
    • 总计Token：35个
    （统计已重置，下一轮对话重新开始计数）

用户：（24小时后）今天吃什么？
Bot：建议吃...
Bot：⏰ 定时Token统计（已24.3小时未查看）：
    • 请求次数：3次
    • 输入Token：45个
    • 输出Token：60个
    • 总计Token：105个
    （统计已重置，下一轮定时统计将在24小时后进行）
```


### 配置Schema
```yaml
config_schema:
  type: object
  properties:
    interval_hours:
      type: number
      title: 自动统计间隔（小时）
      description: 超过此时间未使用/token，下一次AI回复后会自动输出统计信息
      default: 24
      minimum: 1
      maximum: 720
```

### 注意事项
1. 自动统计在用户发送消息并得到AI回复后才会触发
2. 如果会话在一段时间内无活动，统计记录会被自动清理（默认72小时，可在插件配置界面调整）
3. 插件设置界面的配置会持久化保存，修改配置后插件会自动重新加载

### 故障排除
- **插件加载失败**：检查Python版本和依赖
- **没有统计记录**：确认LLM提供商返回了usage信息
- **自动统计不触发**：检查间隔时间设置，确认会话未过期
- **配置不生效**：检查配置格式，确认在有效范围内（1-720小时）
